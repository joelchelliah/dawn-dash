# src/codex/ — Cardex, Skilldex, and Eventmaps

This directory hosts three tools: **Cardex** (`cards.tsx`), **Skilldex** (`skills.tsx`), and **Eventmaps** (`events.tsx`), sharing `components/`, `hooks/`, `utils/`, and `types/`.

## Key files
- `components/SearchPanels/` + `components/ResultsPanels/` — per-tool search UI and result rendering (incl. `TalentTree`, `EventTree`)
- `hooks/useSearchFilters/` — filter state + filtering logic (`talentTreeFilter.ts` is concrete/talent-specific by design; don't re-generalize for one consumer)
- `utils/tree/` — infrastructure **shared by talent and event trees**: text-width estimation, node-dimension factory, tree helpers, SVG setup. Fix bugs here once, not per-feature.
- `utils/eventTreeSpacing/` — multi-pass horizontal layout engine for event trees; read its `README.md` before touching it
- `data/event-trees.json` — static event data, regenerated by `scripts/sync-events.js` (never hand-edit)

## Invariants
- **Layout/render split**: tree layout is memoized on tree/filter/formatting inputs; zoom only re-renders. Both tree components are `React.memo`'d and take scalar props, not whole hook-result objects. Don't merge the layout and render effects back together.
- Node-dimension caches are keyed by all rendering settings; the event cache is cleared only on event change/unmount, not per settings change.
- Cards/talents come from Supabase (24h localStorage cache; versioned keys in `codexCardsStore.ts` / `codexTalentsStore.ts`); events come from the static JSON.
- Tree layout/rendering changes are verified **visually** (before/after comparison in the dev server, incl. expanded/collapsed nodes and zoom levels), not with unit tests.
